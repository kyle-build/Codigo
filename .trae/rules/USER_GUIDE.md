# 协作与风格（精简版）

1. 称呼我为“小黑”
2. 凡涉及逻辑错误、架构违规或代码回退：必须在 `.trae/retros/BAD_CASES.md` 复盘（问题现象 / 违规根因 / 修正方案）
3. 前端遵循 React + Tailwind CSS；禁止引入未声明的第三方 UI；组件深度 ≤ 3
4. 实体与类型准入：严禁随意新增 Entity；新增类型先检索 `types/` 与 `models/`，优先复用
5. 上线前自审（Pre-flight）
   - 安全：无未清理定时器/监听；不输出敏感信息
   - 解耦：业务逻辑不渗入 UI；BroadcastChannel 跨 Tab 同步保持封闭
   - 性能：避免不必要的组件重渲染
6. 需求不清楚先问清再动手
7. 跨包修改记得 build / typecheck
8. 文件建议规模（超出优先拆分职责）
   - `components/`：30-80（>150 必拆）
   - `hooks/`：20-60（>100 建议拆）
   - `pages/`：50-120
   - `store/`：50-150
   - `services/api/`：50-200
   - `utils/`：10-50
   - `types/`：20-100
   - `styles/`：100-300
9. 新函数用文档注释标注
